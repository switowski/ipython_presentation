%magic

Also, IPython comes with a bunch of magic functions built it. Yes, they are seriously called 'magic function'. Let's see what we can do with that. Let's write a simple function. And by write, I mean copy paste it from the next slide.

%edit


def hello(name):
    print 'Hello', name '!'
What happens if I paste it into IPython ? Well, I get an error, cause I forgot the comma. I could press arrow up and try to fix it, but when the function has multiple lines this can become quite a hassle. So we can use another magic function: %edit. If will open your favorite text editor (or Vim, if you haven't configured it) and allow you to edit your code there. After you save and exit, it will execute the code. But if has one great advantage, if you run if with -p option, it will reopen the same file as last time. So you can fix the errors.

%debug


def hello(name):
    print 'Hello', name, '!'
    print number + 42
But what happens if the function is more complicated and I don't see the error right away ? Well, one option is to stare at the code a bit longer until you get the error, but the easier way is to use the %debug function. It will automatically find the error in the code and fix it for you. How cool is that ? Well, no, it won't do that (yet!), instead it will start the debugger session in the place where the error has occurred, so you can poke around and find the error.

%cpaste


def hello(name):
    print 'Hello', name, '!'
    print number + 42
number = 0
So, now we have fully working function. Except that if you try to copy it into IPython, it will complain about indentation error. And yes, IPython has a magic function to fix that, it's called %cpaste and it will automatically adjust the indentations of all the code you paste. 
%timeit function()
%history 10-20 ~1/1,5,7
%pastebin 10-20
# 'https://gist.github.com/8f03e623fe366'
%macro foo 10-20
%save foo.py 10-20
There are few more interesting functions of IPython: * you can use %timeit to measure the execution time of a function * with %history you can print previous commands, for example this will print lines 10 to 20 from the current IPython session and line 1st, 5th and 7th from previous IPython session. * if you think some of the lines are worth sharing, you can use %pastebin command. It will save lines 10-20 on the pastebin and return the URL that you can share with someone. * you can save some commands as a macro with %macro command. This command will save lines 10-20 as a macro, so each time you type foo, it will execute those 10 lines. * and if you are particularly happy with some code you just wrote, you can save it to a file with %save command. This example with save lines 10-20 in a file called foo.py.

Debugging your code with IPython: 

some code
import ipython; ipython.embed()
some other code that fails

My last piece of advice is that you can use IPython for debugging any of your scripts. Just put this command anywhere in your code and if the code execution reaches this place, it will open IPython session in your terminal. Works very well for debugging websites that you have hosted locally.
